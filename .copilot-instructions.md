# SHNGear Development Guidelines

## 🔴 CRITICAL: Frontend API Mapping Rule

**When implementing frontend code that consumes backend APIs, you MUST:**

1. **Read the backend controller endpoint first**
   - Locate the Controller method (e.g., AccountController, ProductController)
   - Understand the return type: `Ok(new ApiResponse<T>())` or raw data?
   - Trace the actual response structure

2. **Test backend response with Postman or PowerShell**
   - Do NOT assume response structure from type declarations
   - Actually call the endpoint and inspect the JSON
   - Document the full response including nesting, field names, and data types

3. **Match Frontend Types to Actual Backend Response**
   - Admin endpoints (Account/Role/Permission): Wrapped in `ApiResponse<T>`
     - Access via: `response.data.data` (for arrays)
     - Example: Account list returns `{ data: [...] }` wrapped in ApiResponse
   - Product endpoint: Returns raw `PagedResult<T>` (NO ApiResponse wrapper)
     - Access via: `response.data.items` (direct to items array)
     - This is the EXCEPTION - different from other admin endpoints

4. **Verify Before Committing**
   - Test FE component with backend running
   - Check browser console for runtime errors (especially `n.map is not a function`)
   - Confirm data displays correctly on the page

## Why This Matters

Previous mistakes caused by skipping backend verification:
- ❌ `response.data.data.items` → should be `response.data.items` for products
- ❌ `.map()` called on objects instead of arrays
- ❌ 404 errors from duplicate `/api` prefixes
- ❌ Type mismatches causing runtime failures

## Backend Response Patterns Reference

| Endpoint Type | Response Wrapper | Access Pattern | Example |
|---|---|---|---|
| Account API | `ApiResponse<T>` | `response.data.data` | `response.data.data[0].email` |
| Role API | `ApiResponse<T>` | `response.data.data` | `response.data.data[0].name` |
| Permission API | `ApiResponse<T>` | `response.data.data` | `response.data.data[0].code` |
| Product API | `PagedResult<T>` | `response.data.items` | `response.data.items[0].name` |
